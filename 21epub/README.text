
新框架

介绍
	新框架是遵循AMD规范开发的一套跨平台的js框架。
	核心思想：
		通过模块化，将现有的功能进行解耦，提高模块的重复利用率和降低模块的错误率
	设计理念：
		一分代码，多平台使用
功能：
－－支持像python一样的同步导入模块功能（以完成） 
－－自动处理模块间依赖关系（已完成）
－－上线前js打包功能（已完成）
－－上线前css打包功能（未完成）
－－上线前图片优化（未完成）
－－上线前js校验（未完成）
－－自动处理不同终端
	－－pc（已完成）
	－－mobile(未完成)
－－页面数据采集（未完成）
－－页面js错误监控（未完成）
－－本地／开发环境切换（未完成）

目录结构：
	21epub
		－－app － 存放业务逻辑代码
		－－pc - pc端的库和模块
			－－ modules
			－－ core.js
		－－mobile - mobile端的库和模块
			－－ modules
			－－ core.js
		－－ nodejs - 基于nodejs的一些工具
		－－project － 项目文件
		－－meta.js － 种子文件
使用方法：
html页面：
	加载如下代码：
		<script src="http://pm.21epub.com/xdad/21epub/meta.js" data-main='epub.app.works20130101'></script>
		－－http://pm.21epub.com/xdad/21epub/meta.js为种子文件地址
		－－data-main指向业务逻辑js文件
业务逻辑js
	eg：works20130101.js
	//页面代码都放入epub.app中，
	epub.app(function (){
		//同步导入需要的模块
		epub["import"]('epub.modules.tmpl@1.0')
		//异步导入需要的模块
		epub["import"]('epub.modules.tmpl@2.0',function(){

		})
	})
模块命名方式介绍
	eg: epub.modules.tmpl@2.0

		－－ epub －整个框架的包名，所有模块都基于这个包
		－－ modules － 模块包，存放所有的模块
		－－ tmpl - 模块名称
		－－ 2.0 － 模块的版本号
模块定义
	
	epub.def(function(){
		epub["import"]('epub.modules.tmpl@1.0')
	})
	如果存在依赖的包，模块需要放入epub.def中定义
	若无依赖关系，则可像jquery插件一样，直接定义
